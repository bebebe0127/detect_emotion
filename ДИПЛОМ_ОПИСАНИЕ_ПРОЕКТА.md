# Подробное описание проекта (для диплома)

## 1. Название и назначение

**Название (рабочее):** система распознавания эмоций по изображению лица (Emotion Recognition).

**Назначение проекта:** обучение и применение нейросетевой модели, которая по изображению лица человека определяет одну из базовых эмоций. Проект включает:

- подготовку датасета изображений эмоций и разбиение на обучающую/валидационную/тестовую выборки;
- обучение сверточной нейронной сети (CNN) на изображениях 48×48 (градации серого);
- запуск инференса в реальном времени с веб‑камеры: детекция лица и классификация эмоции с выводом результата на кадр.

**Практическая ценность:** распознавание эмоций может использоваться в HCI (человеко‑машинное взаимодействие), аналитике пользовательского опыта, образовательных системах, мониторинге состояния оператора, маркетинговых исследованиях и др.

## 2. Что именно реализовано в коде (по файлам)

### 2.1. Подготовка данных

Файл: `src/data/prepare_dataset.py`

- Исходный датасет ожидается в каталоге `src/data/raw/test/`, внутри которого есть подпапки‑классы: `angry`, `disgust`, `fear`, `happy`, `neutral`, `sad`, `surprise`.
- Скрипт собирает пути ко всем изображениям и метки классов, затем делает стратифицированное разбиение:
  - \(70\%\) — `train`
  - \(15\%\) — `val`
  - \(15\%\) — `test`
- После разбиения изображения **копируются** в структуру `data/processed/<split>/<label>/...`.

Фактические размеры `data/processed` (по текущим данным в вашем проекте):

- `train`: 5024 изображений  
  `{'angry': 670, 'disgust': 78, 'fear': 717, 'happy': 1242, 'neutral': 863, 'sad': 873, 'surprise': 581}`
- `val`: 1077 изображений  
  `{'angry': 144, 'disgust': 16, 'fear': 154, 'happy': 266, 'neutral': 185, 'sad': 187, 'surprise': 125}`
- `test`: 1077 изображений  
  `{'angry': 144, 'disgust': 17, 'fear': 153, 'happy': 266, 'neutral': 185, 'sad': 187, 'surprise': 125}`

### 2.2. Загрузчики данных (генераторы)

Файл: `src/data/dataloaders.py`

- Используется `ImageDataGenerator` из Keras с нормализацией пикселей `rescale=1./255`.
- Загрузка данных идёт через `flow_from_directory` для каталогов:
  - `data/processed/train`
  - `data/processed/val`
- Приведение изображений:
  - размер `target_size=(48, 48)`
  - `color_mode="grayscale"` (один канал)
- Выходной формат меток: `class_mode="categorical"` (one‑hot).

> Важно для диплома: в Keras порядок классов по умолчанию определяется **алфавитно по именам подпапок**. В вашем датасете это обычно:  
> `['angry', 'disgust', 'fear', 'happy', 'neutral', 'sad', 'surprise']`.  
> Этот порядок должен совпадать с порядком меток, которые вы используете при показе результата на видео.

### 2.3. Архитектура модели (CNN)

Файл: `src/models/cnn_baseline.py`

Модель — последовательная CNN для классификации на 7 классов:

- `Conv2D(64, 3×3) + ReLU` → `MaxPool(2×2)`
- `Conv2D(128, 3×3) + ReLU` → `MaxPool(2×2)`
- `Conv2D(256, 3×3) + ReLU`
- `Conv2D(256, 3×3) + ReLU` → `MaxPool(2×2)`
- `Flatten`
- `Dense(512) + ReLU` → `Dropout(0.5)`
- `Dense(7) + Softmax`

**Вход модели:** изображение \(48 \times 48 \times 1\).  
**Выход:** вероятности по 7 классам (Softmax).

### 2.4. Обучение модели

Файл: `src/train.py`

- Берёт генераторы из `get_generators("data/processed")`.
- Создаёт модель `create_model()`.
- Компиляция:
  - оптимизатор: `Adam`
  - функция потерь: `categorical_crossentropy`
  - метрика: `accuracy`
- Обучение `epochs=30`.
- Сохранение в файл: `results/models/emotion_cnn.h5`.

### 2.5. Инференс в реальном времени (веб‑камера)

Файл: `src/inference.py`

Пайплайн:

1. Захват кадров с камеры `cv2.VideoCapture(0)`.
2. Перевод кадра в grayscale.
3. Детекция лиц классическим каскадом Хаара:
   - `haarcascade_frontalface_default.xml` из OpenCV.
4. Для каждого найденного лица:
   - вырезание ROI (область лица),
   - ресайз до 48×48,
   - нормализация \([0..1]\),
   - преобразование в тензор,
   - предсказание `model.predict`.
5. Отрисовка прямоугольника и текстовой метки эмоции на кадре.

Файл: `detect_emotion.py`

- Делает похожий инференс, но загружает модель из `emotion_detection_model.h5` (такого файла в `results/models/` нет).
- Для корректной работы «из коробки» в вашем проекте ориентируйтесь на `src/inference.py` и путь `results/models/emotion_cnn.h5`.

## 3. Теоретическая часть (как описывать в дипломе)

### 3.1. Постановка задачи

**Вход:** изображение лица человека (или ROI лица на видеокадре).  
**Выход:** класс эмоции из множества из 7 категорий.

Задача формулируется как **многоклассовая классификация**:

\[
\hat{y} = \arg\max_k \; p(y=k \mid x),
\]

где \(x\) — входное изображение, \(p(\cdot)\) — распределение вероятностей после слоя Softmax.

### 3.2. Почему CNN

Сверточные сети:

- эффективно извлекают локальные признаки (границы, текстуры, мимические паттерны);
- устойчивее к небольшим смещениям и шуму;
- являются стандартным базовым подходом для задач компьютерного зрения.

### 3.3. Функция потерь

Для one‑hot меток используется категориальная кросс‑энтропия:

\[
L = -\sum_{k=1}^{K} y_k \log(\hat{p}_k),
\]

где \(K=7\), \(y_k\) — истинная метка, \(\hat{p}_k\) — предсказанная вероятность класса \(k\).

## 8. Важные технические замечания (чтобы диплом был честным и сильным)

### 8.1. Порядок классов и отображение эмоций

В `src/inference.py` и `detect_emotion.py` список `emotion_labels` задан вручную. Но Keras при обучении назначает индексы классов по папкам (обычно по алфавиту).

**Риск:** если порядок `emotion_labels` не совпадает с `class_indices` при обучении, подписи эмоций на видео будут неправильными.

**Как правильно (идея для улучшения/описания в дипломе):**

- сохранить `train_gen.class_indices` в JSON при обучении;
- при инференсе загружать этот JSON и строить список меток в правильном порядке индексов.

### 8.2. Источник данных в `prepare_dataset.py`

Скрипт использует `RAW_DIR = "src/data/raw/test"`. В идеале стоит:

- либо использовать объединение `raw/train` и `raw/test`,
- либо явно описать, почему берётся именно этот поднабор (например, это полный набор изображений в вашем случае).

### 8.3. Дублирование инференса

Есть два инференс‑скрипта: `detect_emotion.py` и `src/inference.py`. Для диплома лучше оставить один «главный» сценарий и описывать его как итоговое приложение.

## 10. Идеи для “Дальнейшего развития” (можно включить в диплом)

- улучшение качества: data augmentation, batch normalization, early stopping, learning rate scheduling;
- переход на более сильную архитектуру (например, MobileNet/ResNet) и/или transfer learning;
- замена Haar Cascade на более устойчивую детекцию лиц (DNN‑детектор, MediaPipe, RetinaFace);
- сохранение и загрузка корректного соответствия индексов классов;
- добавление `evaluate.py` и автоматической генерации отчёта (метрики + confusion matrix);
- оптимизация инференса (ускорение на GPU, квантование, TensorRT/ONNX).
